What is ISO 15939?

Fundamentals of the Measurement Process

Definition

ISO/IEC 15939 is an international standard that defines the measurement process in software and systems engineering.

üéØ Purpose

To establish a measurable, repeatable, and reliable measurement process in software projects

üìä Scope

The entire process from identifying information needs to interpreting indicators

üîó Integration

Used together with ISO 25010 (quality model) and ISO 25023 (quality measurements)

‚úÖ Benefits

Objective evaluation, data-driven decisions, continuous improvement

üí° Why is it Important?

"You can't manage what you can't measure" - Peter Drucker

ISO 15939 provides the ability to evaluate software quality with objective data.

Process Logic

  • Top-Down: Start from needs and descend to concrete measurements
  • Structured: Each step uses the output of the previous step
  • Repeatable: Standard process can be used in every project
  • Result-Oriented: Indicators support decision making
üîç Practical Example: The journey from "Do our users find our software usable?" to a usability score

Step 1: Information Need

Defining the Purpose of Measurement

üìã Definition

The stage of determining the purpose of measurement and what information is needed.

Question: "What do we want to learn about?"

Information Need Example Description
What is the usability level of the software? User experience evaluation
Does the system have security vulnerabilities? Security status assessment
Have we met our performance targets? Performance evaluation
Is code quality sufficient? Maintainability analysis

Well-Defined Information Need

  • In the form of a clear and specific question
  • Aligned with decision makers' needs
  • Can be converted to measurable concepts
  • Consistent with project objectives

Step 4: Base Measure

Direct Measurement of Raw Data

üìè Definition

Raw data obtained through direct measurement of entity attributes.

Question: "What raw data will we collect?"

Base Measure Characteristics

  • Requires no mathematical operations
  • In raw data format
  • Measures a single attribute

Collection Methods

  • Log file analysis
  • Automated measurement tools
  • User testing
  • System monitoring

Practical Application Example

Mobile Application Quality Assessment

üéØ 1. Information Need
